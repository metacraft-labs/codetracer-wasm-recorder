# Coding agents instructions

## Agent Knowledge Log

The file `.agents/code-insights.md` is your checklist before interacting the codebase. Treat it as mandatory reading and maintenance.

- **Before starting any task, ALWAYS open and review** `.agents/code-insights.md` so you inherit the latest context.
- Capture new insights, architectural notes, domain knowledge, debugging breadcrumbs, surprising findings, edge cases, and other non-trivial behaviors in `.agents/code-insights.md`.
- Keep entries in that insights file clear, concise, and dated when helpful so others can trust the context quickly.
- Update the insights file continuously: add new learnings immediately, revise stale items, and REMOVE information the moment it stops being true.
- If you are unsure whether something belongs in `.agents/code-insights.md`, err on the side of writing it down.

Maintaining `.agents/code-insights.md` is part of completing every task.

## Testing Standards

- Always design and commit comprehensive tests that cover baseline behavior and all edge cases **before** implementing the associated functionality.
- Consider a task complete only after all relevant tests run and pass.

## Code Comments

- Write detailed comments for new code only when the intent would not be obvious to someone unfamiliar with the change.
- Keep existing comments accurate by updating or pruning them when related behavior changes.
- Explain any non-trivial, complex, or surprising behavior so future maintainers understand why the code exists.
- Avoid comments that restate widely understood facts; instead, craft code that is self-documenting with clear names, structure, and small, focused functions.

## Code Quality

- ALWAYS strive for high-quality, secure code that embraces best practices and keeps future maintenance in mind.
- Respect DRY, KISS, and the Single Responsibility Principle: avoid duplication, keep solutions as simple as the requirements allow, and resist speculative abstractions while ensuring each unit has a focused purpose.
- Build fault-tolerant paths with appropriate error handling, clear failure messages, and graceful recovery when practical.
- Favor reusable, flexible designs through clear boundaries, small composable units, and dependency injection when it improves testability.
- Enforce least-privilege access, validate inputs, and stay vigilant against common security pitfalls.
- Maintain readability with consistent style, expressive names, and documentation only where the code itself cannot be clearer.
- Balance performance with clarity; measure before optimizing and back changes with tests or benchmarks when performance matters.
- Always format and lint new or modified Go code with `go fmt`, `go vet`, `golangci-lint fmt`, `golangci-lint run`, and `staticcheck` before calling the work complete. Resolve every reported issue or have a documented justification in a comment or in `.agents/code-insights.md` for any intentional exception.

## Libraries & Dependencies

- Use libraries when they provide reliable, tested functionality and reduce boilerplate.
- Prefer the standard library first whenever it covers the need effectively.
- Only introduce external dependencies when:
    - they provide significant value.
    - They are well-maintained, widely used, and trustworthy.
- Document in .agents/code-insights.md when a dependency choice.

## Commit Messages

- Structure every commit message as multiple lines: a first-line summary followed by one or more detail lines and a closing attribution line.
- Ensure the first line follows the Conventional Commits format.
- Use the body lines after the summary to concisely describe the key changes included in the commit.
- Always end the message with the exact line `Generated by $AGENT_NAME` where $AGENT_NAME is the name of the agent that generated the code/commit (for example `Generated by Codex`).

### Example commit message
```
chore: document agent workflow setup
- Add agent protocol docs and code insight checklist files.
- Update gitignore and nix environment to align with new tooling.
- Refresh flake lock metadata.

Generated by Codex
```
